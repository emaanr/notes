# `Wrapper` Class

- Provides a way to use primitive data types as reference data types.
- Reference data types contain useful methods.
  - Can be used with certain collections such as `ArrayList`.

# Autoboxing

- The automatic conversion that the Java compiler makes between the primitive type and their corresponding object wrapper class.

# Unboxing

- The reverse of autoboxing.
- Automatic conversion of wrapper class to corresponding primitive data type.

# Primitive vs Reference

| Primitive | Wrapper   |
| --------- | --------- |
| boolean   | Boolean   |
| char      | Character |
| int       | Integer   |
| double    | Double    |

- Some examples are provided in the table above.
  - Note that `String`s are not present because they are already reference data types.

> Note that as previously discussed, there are tradeoffs for using primitive data types or reference data types, including but not limited to:
>
> - Primitive data types are faster.
> - Reference data types grant access to useful methods.
